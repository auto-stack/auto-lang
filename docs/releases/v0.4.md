## 版本内容

v0.4版在v0.3的基础上，重点突破了解释器运行时的全功能化、a2r转译器的生产级可用性、以及AI Agent基础设施的搭建。

### 一、AutoVM 解释器运行时（重大突破）

v0.3的AutoVM只能运行简单的脚本和测试用例。v0.4让AutoVM具备了**运行真实应用后端**的能力。

1. **#[api] HTTP服务器**（Plan 312, 313, 321, 322）：
   - `#[api]` 注解自动生成HTTP路由（GET/POST/PUT/DELETE）
   - 路径参数提取（`:id`）、JSON序列化（结构体/数组/Option）
   - SSE流式响应（`~Iter<T>`/`~Stream<T>` → SSE frames）
   - TCP nodelay + flush保证SSE实时性
   - 检测到路由时自动启动HTTP服务器

2. **异步调度统一**（Plan 317）：
   - Task/Msg actor 处理程序真实执行（VM内部调度，path B）
   - `LOAD_STATE_FIELD`/`STORE_STATE_FIELD`：任务间持久化的actor状态
   - 惰性生成器：`next()` 运行到下一个 `YIELD_VAL`，任务在调用间存活
   - 并发HTTP服务器：tokio LocalSet + spawn_local 实现单线程上的 !Send 异步
   - Goroutine风格协作调度：单worker + yield_now 实现SSE交叉流

3. **多模块支持**（Plan 317 Phase B）：
   - 真正的 `use` 模块加载（CompileSession + Linker），替代字符串展平
   - 允许循环模块依赖（db use api + api use db）
   - `LOAD_GLOBAL`/`STORE_GLOBAL`：跨 `#[api]` 处理程序持久化的模块级变量
   - 32MB大栈线程避免解析器/代码生成栈溢出

4. **生成器与迭代器**（Plan 321）：
   - `yield` 表达式 + `YIELD_VAL` 操作码 + `Iterator::Generator`
   - 生成器for循环修复（nil哨兵防止重复值）
   - `~Iter<T>`/`~Stream<T>` 统一为HTTP Iter协议

5. **List<T> 运行时**：
   - `List<自定义结构体>` 泛型解析（如 `List<Note>`）
   - `List<T>.new`、`.push`、`.filter`、`.to_array` 运行时支持
   - 数组fallback + Value透传

6. **结构体/枚举运行时**（Plan 326）：
   - 结构体字段访问（`note.title`）
   - HTTP序列化（结构体/数组/Option → JSON）
   - int路径参数转换
   - 记录类型别名（`type Name = { fields }`）+ 冒号字段语法

### 二、a2r 转译器（Auto → Rust）成熟

v0.3的a2r只能转译简单的cookbook示例。v0.4让a2r具备**转译并运行AutoForge级别后端**的能力。

7. **Axum HTTP服务器生成**（Plan 328）：
   - 从 `#[api]` 注解生成Axum处理程序 + 路由
   - 共享状态（Arc<Mutex<Db>>）、CORS支持
   - SSE处理程序生成 + 返回类型映射

8. **Tauri IPC支持**（Plan 328, 329）：
   - 生成 `#[tauri::command]` IPC函数
   - Tauri Channel实现SSE流式传输
   - `--render=tauri` 自动建立Tauri工程

9. **逃逸分析引擎**（Plan 310）：
   - EscapeMap + 分析管线
   - View/Mut代码生成按逃逸层级选择 borrow/clone/Rc
   - 自引用检测 + 间接循环警告

10. **a2r转译器核心增强**（Plan 229, 241, 266）：
    - 多文件项目合并转译（merge mode）—— 1427→0 错误
    - AST级修复替代正则后处理
    - String类型系统、auto-borrow、auto-clone for non-Copy
    - VM ↔ a2r 一致性测试框架（30个conformance测试）

### 三、Shell生态（ASH）—— 从玩具到日用

v0.4的ASH（Auto Shell）从一个简单的REPL成长为**可日常使用的现代Shell**。

11. **ASH核心功能**（Plan 291-304, 318, 320, 322）：
    - 任务控制：`cmd &`、`jobs`、`fg`、`bg`、Ctrl+C保护
    - 管道与流：OS pipe链式、stdin piping、实时输出、外部命令检测
    - 补全系统：ratatui表格渲染、模糊匹配、任意命令补全、help/man解析
    - 模式系统：F1/F2/F3模式切换、多行续行、保守Auto表达式检测
    - 配置系统：`config.at`（Auto/Atom主配置）+ `prompt.at`（提示符配置）
    - 流水线DSL：`filter`/`sort`/`select`/`map`/`take`/`count` 聚合与复合谓词

12. **24-bit真彩色**（Plan 317）：
    - 真彩色检测 + 降采样
    - 24-bit语法高亮主题
    - `color rainbow`/`color depth` 演示

13. **环境变量系统**（Plan 301, 309）：
    - `env`/`env.path` 命名空间、PATH一等List、作用域隔离
    - `~user` 展开、`did-you-mean` PATH扫描
    - brace展开、算术展开、`$?`/`$@`/`$#`/`$!` 特殊参数
    - env持久化（`-save`/`-load` + 启动自动加载）

## Task 1: P0 — Option/Result 支持

**映射策略：**

| Auto 构造 | TypeScript 输出 |
|-----------|----------------|
| `Option<T>` | `T \| null` |
| `Some(x)` | `x` |
| `None` | `null` |
| `Result<T, E>` | `T \| Error` |
| `Ok(x)` | `x` |
| `Err(msg)` | `throw new Error(msg)` |
| `x ?? default` | `x ?? default`（完美映射！） |
| `x?.method()` | `x?.method()`（完美映射！） |

**注意：** TypeScript 的 `??` 和 `?.` 操作符与 Auto 完全对应，这是 a2ts 相比 a2py 的巨大优势——无需特殊转换。

**创建测试：**

```
test/a2ts/09_option_result/001_option/option.at:
fn main() {
    let x = Some(42)
    let y = None
    let val = x ?? 0
    print(val)
    let name: ?str = None
    let result = name?.len()
    print(result)
}
```
```
test/a2ts/09_option_result/001_option/option.expected.ts:
// Auto-generated by a2ts transpiler

const print = console.log.bind(console);

function main(): void {
    const x: number | null = 42;
    const y: null = null;
    const val = x ?? 0;
    print(val);
    const name: string | null = null;
    const result = name?.length;
    print(result);
}

main();
```

## Task 5: P1 — 泛型约束

**映射策略：**

```auto
fn first<T>(list [T]) T {
    list[0]
}
```
→
```typescript
function first<T>(list: T[]): T {
    return list[0];
}
```

带约束：

```auto
fn compare<T>(a T, b T) int where T: Comparable {
    a.compare(b)
}
```
→
```typescript
function compare<T extends Comparable>(a: T, b: T): number {
    return a.compare(b);
}
```

## Task 6: P1 — 高级模式匹配

**映射策略：**

```auto
is x {
    0 -> print("zero")
    n if n > 10 -> print("big")
    _ -> print("other")
}
```
→
```typescript
switch (x) {
    case 0: print("zero"); break;
    default:
        if (x > 10) { print("big"); break; }
        print("other"); break;
}
```

带 guard 的模式匹配在 TS 中没有完美对应（switch 不支持 guard），需要用 if/else if 链替代。

**创建 3 个测试，提交。**

```bash

```

## Task 8: P2 — async/await

**映射策略：**

```auto
fn fetch(url str) ~str {
    // async body
}
```
→
```typescript
async function fetch(url: string): Promise<string> {
    // async body
}
```

- `~T` 返回类型 → `Promise<T>` + `async`
- `.await` → `await`
- `~{ ... }` async block → `(async () => { ... })()`

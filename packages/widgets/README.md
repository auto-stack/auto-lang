# @auto-ui/widgets

AutoUI-generated Vue 3 component primitives — self-contained `.vue` files backed
by [reka-ui] and styled with [Tailwind CSS]. The components are **generated by
the AutoLang compiler** from AURA widget definitions, not hand-written.

> Visual layer derived from [shadcn-vue] (MIT). See [NOTICES](./NOTICES).

## Install / usage (shadcn-style copy)

```bash
# copy a widget into your project at src/components/ui/<widget>/
npx @auto-ui/widgets add button

# list available widgets
npx @auto-ui/widgets list
```

`add` copies the `.vue` (+ support files) into your project and, by default,
auto-installs `reka-ui` if it's missing. Flags:

- `--no-install` — don't auto-install `reka-ui`.
- `--reka-ui <pkg>` — use a custom fork; rewrites `from 'reka-ui'` imports.
- `--no-styles` — skip the Tailwind / precompiled-CSS guidance.
- `--out <dir>` — override the destination (default `src/components/ui`).

## Styling: pick exactly one path

These widgets use Tailwind utility classes. You have two mutually exclusive
options — do **not** mix them (duplicate `.flex` etc. will conflict):

- **Path A (zero config):** import the precompiled stylesheet once in your app
  entry, and don't run your own Tailwind:
  ```ts
  import '@auto-ui/widgets/styles.css'
  ```
- **Path B (your own Tailwind):** if your project already uses Tailwind, point
  its `content` glob at the copied components (e.g.
  `./src/components/ui/**/*.{vue,ts}`) and do **not** import our `styles.css`.

## Peer dependencies

- `vue` (^3.4) — required peer.
- `reka-ui` (^2.0) — optional peer; the CLI installs it for you unless you pass
  `--no-install` or `--reka-ui`.

## Regenerating the registry (maintainers)

The committed `registry/` is the publish artifact. To regenerate from the
AutoLang compiler:

```bash
# from the repo root, after `cargo build -p auto`
auto ui build --target vue --out packages/widgets/registry
```

Then rebuild the precompiled stylesheet (see Phase 6 build script) and bump the
version before `npm publish`.

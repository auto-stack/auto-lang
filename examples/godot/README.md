# Auto в†’ GDScript Godot Examples

This directory contains game-oriented examples that prove Auto-generated GDScript can run in the Godot 4.x engine. Each example is a complete, self-contained Godot project.

## How to Test

### Prerequisites

- [Godot 4.x](https://godotengine.org/download) (4.2 or later)
- AutoLang compiler built (`cargo build -p auto`)

### Option A: Open in Godot Editor

1. Open Godot Editor
2. Click "Import" в†’ Browse to `examples/godot/001-hello/project.godot`
3. Click "Import & Edit"
4. Press **F5** (or click в–¶ Run) to run the project
5. Check the **Output** panel at the bottom for printed messages

### Option B: Rebuild GDScript from Auto Source

```bash
# From the auto-lang project root

# Build the compiler
cargo build -p auto

# Transpile a specific example
./target/debug/auto trans --path examples/godot/001-hello/src/main.at gd
mv examples/godot/001-hello/src/main.gd examples/godot/001-hello/scripts/main.gd

# Then open in Godot and run
```

### Option C: Command Line (Headless)

```bash
# If Godot is in your PATH
godot --path examples/godot/001-hello --headless --quit
# Check console output for printed messages
```

## Auto в†’ GDScript Mapping Reference

| Auto Syntax | GDScript Output | Notes |
|---|---|---|
| `fn main() { ... }` | `func _ready(): ...` | Entry point в†’ Godot lifecycle |
| `let x = 10` | `var x: int = 10` | Immutable в†’ typed variable |
| `var x = 10` | `var x: int = 10` | Mutable в†’ typed variable |
| `fn foo(a int, b int) int { ... }` | `func foo(a: int, b: int) -> int: ...` | Functions with types |
| `if/else if/else` | `if/elif/else:` | Conditional branching |
| `for i in 0..10` | `for i in range(0, 10):` | Range loop |
| `for cond { ... }` | `while cond: ...` | Conditional loop |
| `type Foo { x int }` | `class_name Foo` + `var x: int` | Type definitions |
| `print(x)` | `print(x)` | Direct passthrough |

## Current Limitations

These examples work within current a2gd capabilities. The following Godot features are **not yet supported** and are planned for Phase B:

| Feature | Status | Needed For |
|---|---|---|
| `extends Area2D/CharacterBody2D/etc.` | вќЊ Always `extends Node` | Player physics, collisions |
| `signal` declarations | вќЊ Not transpiled | Game events |
| `@export` annotations | вќЊ Not transpiled | Editor-exposed properties |
| `$NodePath` syntax | вќЊ Not transpiled | Scene tree access |
| `_process(delta)` | вљ пёЏ Works as function name, but `fn main()` в†’ `_ready()` | Per-frame updates |
| `Vector2` / Godot built-in types | вќЊ Not mapped | Position, velocity, physics |
| `Input.is_action_pressed()` | вќЊ Not mapped | Keyboard/gamepad input |
| `is` as expression returning value | вљ пёЏ Generates `match` (GDScript statement) | Pattern matching with return |
| Type constructor `Stats(a, b, c)` | вљ пёЏ Generated but may need `_init()` | Struct instantiation |
| `await $Timer.timeout` | вљ пёЏ Basic await works | Async game logic |
| `queue_free()` / `add_child()` | вќЊ Not mapped | Scene tree manipulation |

## Roadmap to Full "Dodge the Creeps"

The official Godot tutorial ["Your First 2D Game"](https://docs.godotengine.org/en/stable/getting_started/first_2d_game/index.html) creates a complete "Dodge the Creeps" game with 4 scripts: `player.gd`, `mob.gd`, `main.gd`, and `hud.gd`. Adapting this to Auto requires the Phase B features above.

**Priority order for Phase B:**
1. **Custom `extends` types** вЂ” allow `extends Area2D` instead of always `Node`
2. **`@export` / `@onready` annotations** вЂ” Godot property system
3. **`$NodePath` syntax** вЂ” scene tree access
4. **`signal` system** вЂ” event communication between nodes
5. **Godot built-in types** вЂ” `Vector2`, `Color`, `PackedScene`, etc.
6. **`_process(delta)` mapping** вЂ” separate from `_ready()`
7. **Godot API functions** вЂ” `Input`, `get_tree()`, `randf_range()`, etc.
